# CycleLink Admin Dashboard

Web app for CycleLink admin: usage statistics, sponsor customisation of advertised scenic locations, and developer health/usage monitoring. Intended to be hosted on **Amazon S3** and served via **CloudFront** (see Prelim Report).

## Status

Placeholder only. Add your preferred stack (e.g. React + Vite, or static HTML) when the team is ready. Suggested structure:

- **Usage stats** — sign-up rates, ride completion, post-ride satisfaction, 7/30-day return.
- **Sponsor admin** — manage advertised waypoints / scenic locations.
- **Developer dashboard** — cluster uptime, API latency, route generation latency.

Backend endpoints for these are to be defined with the backend team; frontend can mock data until then.
